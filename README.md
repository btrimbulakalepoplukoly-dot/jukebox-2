# Chaos Jukebox — мод для Minecraft 26.2 "Chaos Cubed" (Fabric)

Блок, який крафтиться з **4 земель** (2×2), і по **ПКМ** (правий клік) відкриває
вікно в стилі Minecraft, де можна вставити посилання на **YouTube**. Звук/музика
з відео починає грати прямо з блока, чутно в радіусі **30 блоків**, з динамічним
затуханням гучності залежно від відстані.

## 🚀 Найшвидший спосіб: один bat-файл

У цій папці лежить `build_and_install.bat` (+ `download_fabric_api.ps1` поруч,
не видаляй). Просто двічі клікни на `build_and_install.bat` (або запусти його
з `cmd`). Він сам:

1. перевірить Java, поставить Git, якщо нема;
2. склонує офіційний шаблон Fabric для 26.2 в підпапку `build-workspace`;
3. скопіює туди код мода з цієї папки;
4. зробить `gradlew.bat build`;
5. покладе готовий `.jar` у `%appdata%\.minecraft\mods`;
6. сам знайде й завантажить актуальний Fabric API 26.2 (через Modrinth);
7. встановить `yt-dlp` і `ffmpeg`, якщо їх ще нема.

Потрібен лише інтернет і трохи терпіння на першу збірку (кілька хвилин).
Якщо щось піде не так — скрипт зупиниться і напише, що скопіювати мені для
діагностики.

Усе нижче — те саме, але руками, якщо хочеш розуміти кожен крок або bat-файл
чомусь не підійде.

## ⚠️ Важливо прочитати перед збіркою (ручний спосіб)

1. **Це не скомпільований jar-файл, а вихідний код проєкту.** Я не маю доступу до
   серверів Mojang/Fabric у цьому середовищі, тому не можу сам зібрати й
   протестувати мод. Тобі потрібно зібрати його на своєму ПК (звичайний
   `./gradlew build`, інтернет потрібен лише для першої збірки — Gradle сам
   скачає Minecraft/Fabric).
2. Версії в `gradle.properties` (yarn mappings, loader, fabric-api) актуальні
   станом на момент написання, але Fabric для 26.2 оновлюється дуже часто
   (релізу всього ~2 тижні). **Якщо Gradle не може щось знайти** — зайди на
   https://fabricmc.net/develop/template/, обери Minecraft `26.2`, згенеруй
   шаблон і просто скопіюй з нього актуальні рядки в `gradle.properties` /
   `build.gradle`. Увесь Java-код і ресурси з цього проєкту просто копіюються
   в згенерований шаблон.
3. **Відтворення YouTube вимагає двох зовнішніх програм, встановлених на ПК
   гравця (того, хто чує звук):**
   - [`yt-dlp`](https://github.com/yt-dlp/yt-dlp) — дістає пряме посилання на
     аудіопотік відео.
   - [`ffmpeg`](https://ffmpeg.org/download.html) — перетворює цей потік у
     звук, який можна грати.

   Обидві мають бути в `PATH` (тобто команди `yt-dlp` і `ffmpeg` мають
   працювати з будь-якої папки в командному рядку). Це той самий підхід, який
   використовують музичні боти Discord — мод нічого не "качає" і не вшиває
   YouTube в себе, лише питає в yt-dlp пряму URL і стрімить її.

   Звук рахується й грається **на кожному клієнті окремо**, тож встановити
   yt-dlp/ffmpeg треба кожному гравцю, який має чути музику (не лише власнику
   сервера).

| `network/JukeboxPackets.java` | Пакети клієнт↔сервер (Play/Stop, синхронізація) |

| `assets/.../textures/block/stream_jukebox.png` | **Тимчасова текстура-заглушка** |

## Як це працює (коротко)

1. Гравець ставить блок і тисне ПКМ → відкривається `JukeboxScreen` (тільки
   локально, без затримки на сервер).
2. Вставляє лінк, тисне "Грати" → клієнт відправляє `SetUrlPayload` на сервер.
3. Сервер зберігає URL у `BlockEntity` і розсилає `PlayAudioPayload` усім
   гравцям у світі (з позицією блока, лінком і часом старту).
4. Кожен клієнт окремо: `yt-dlp` дістає пряме аудіо-URL → `ffmpeg`
   перетворює на PCM → грає через `SourceDataLine`.
5. Кожен клієнтський тік `ActiveAudioManager` рахує відстань гравця до блока
   і виставляє гучність: 0 блоків = 100%, 30+ блоків = тиша (лінійне
   затухання).
6. Гравці, що заходять у радіус пізніше, теж почують музику — `ffmpeg`
   запускається з `-ss <скільки секунд уже минуло>`, щоб приблизно
   синхронізувати позицію (не ідеально точно, але достатньо для "фону").

## Заміна текстури на твоє фото

Зараз там тимчасова згенерована іконка-плейсхолдер. Щоб поставити своє фото:

1. Зроби квадратне зображення (наприклад 32×32, 64×64 або навіть 256×256 —
   Minecraft підтримує будь-який квадратний розмір, степінь двійки бажана).
2. Назви файл `stream_jukebox.png` і поклади замість
   `src/main/resources/assets/chaosjukebox/textures/block/stream_jukebox.png`.
3. Якщо хочеш різні картинки на верх/низ/боки — скажи, я перероблю модель з
   `cube_all` на `cube_bottom_top` або повну `cube` з 6 текстурами. Зараз
   модель розрахована саме на **одне фото на всі 6 сторін**, як і просив.

## Чому без прямого WebDisplays/MCEF

Я перевірив: останній Fabric-порт WebDisplays (потребує MCEF — вбудований
Chromium) офіційно є аж до Minecraft 1.20.1, і під 26.2 готової збірки MCEF
ще немає — це дуже важка нативна залежність (цілий браузерний движок), яку
довелось би компілювати окремо під 26.2. Тому в цьому моді відтворення
зроблено напряму через yt-dlp+ffmpeg — це надійніше, легше і не вимагає
встановлення окремого мода-браузера. Якщо пізніше з'явиться стабільний
WebDisplays/MCEF для 26.2 — інтеграцію можна додати (відкривати "браузер" на
екрані замість текстового поля), скажи, і я підготую цю версію.

## Збірка

```bash
./gradlew build
# готовий jar буде в build/libs/chaosjukebox-1.0.0.jar
```

Кидаєш jar у папку `mods` (поряд з Fabric API 26.2 і Fabric Loader 0.19.3+).

## Відомі обмеження (чесно попереджаю)

- Синхронізація між гравцями приблизна (за часом, не за фреймами).
- Якщо `yt-dlp`/`ffmpeg` відсутні — гравець побачить повідомлення в чаті з
  поясненням, музика просто не почне грати (без крашу).
- YouTube інколи змінює свій захист — якщо `yt-dlp` перестав working,
  команда `yt-dlp -U` оновлює його до останньої версії.
- Це перша версія коду, не протестована живою грою (немає доступу до
  Minecraft/Fabric серверів із цього середовища) — можливі дрібні правки
  імен методів під точний білд мапінгів 26.2, дивись розділ вище.
